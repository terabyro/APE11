Principais comandos do Git | sem Flow

Comando Descrição
$ git config –global user.name “Seu nome” O comando git config faz a configuração do usuário na máquina, ao adidionar a flag -global as configurações são aplicadas de forma global. Nesse exemplo configurando o nome do usuário
$ git config –global user.email “Seu e-mail” O comando git config faz a configuração do usuário na máquina, ao adidionar a flag -global as configurações são aplicadas de forma global. Nesse exemplo configurando o e-mail do usuário
$ git init [nome do repositório] Cria um novo repositório com o nome informado logo após o comando
$ git clone [URL do projeto] Faz o clone de um repositório remoto para a máquina através da url do projeto inforada
$ git add * ou git add [nome do arquivo] O primeiro comando adiciona todos os novos arquivos ou arquivos modificados para stage (também é possível utilizar a flag --all), o segundo adiciona um arquivo específico
$ git commit -m “Mensagem do commit” Comando confirma as alterações feitas e adicionadas em stage
$ git push Envia os commits do repositório local para o repositório removo
$ git pull Puxa os arquivos atualizados do repositório removo para o repositório local
$ git branch [nome da branch] Cria uma branch local com base no código da branch que esta com checkout
$ git checkout [nome da branch] Faz o checkout na branch cujo nome foi informado
$ git checkout -b [nome da branch] Esse comando é a união dos dois últimos, ele criar uma nova branch e já o checkout na branch criada
$ git tag -a [v1.0.0] Cria uma tag com a marcação informada, como no exemplo v1.0.0
$ git push --tags Envia as tags do repositório local para o repositório removo
